# sync.Pool vs valyala/bytebufferpool

Для фиксированного 64KB буфера в `PipeCopy` **`sync.Pool` лучше**.

| Критерий       | `sync.Pool`                     | `valyala/bytebufferpool`                 |
|----------------|---------------------------------|------------------------------------------|
| Тип            | `[]byte` (raw)                  | `*ByteBuffer` (структура с `B []byte`)   |
| Оверхед        | Минимальный                     | Дополнительная обёртка, вызов `buf.B`    |
| Размер         | Фиксированный                   | Переменный (растёт при необходимости)    |
| Как в fasthttp | `copyBufPool` (для копирования) | `body` в `Request` (для переменных тел)  |

**Вывод**: fasthttp сам использует `sync.Pool` для копирующих буферов (`copyBufPool`), а `bytebufferpool` — для объектов, которым нужен переменный размер. Для фиксированного 64KB буфера в `PipeCopy` `sync.Pool` — правильный выбор.
