# ChangeLog PR-13 — TLS/HTTPS поддержка для upstream и сервера

## Сводка

Реализована поддержка TLS/HTTPS для upstream-соединений и опционального TLS на стороне прокси-сервера. Добавлены флаги конфигурации, парсинг схем URL (`http://`/`https://`), структурированное хранение upstream-адресов и TLS-соединения через `tls.Client`.

## Ключевые изменения

1. **Схемы URL**: адреса `http://host:port`/`https://host:port` обрабатываются явно; без схемы — `http://` (обратная совместимость).
2. **Upstream TLS**: глобальный `tlsConfig` в pool; `dial()` для `https://` использует `fasthttp.Dial` + `tls.Client`.
3. **Серверный TLS**: `constructTLSConfig` загружает сертификат/ключ из PEM-файлов и устанавливает `s.TLSConfig`.
4. **Upstream struct**: `*Upstream` хранит разобранный адрес, схему, хост, порт; `Random()` возвращает структуру, а не строку.
5. **Новые флаги**: `--tls-enabled`, `--tls-insecure-skip-verify`, `--tls-ca-file`, `--tls-server-certificate-pem-file`, `--tls-server-key-pem-file`, `--tls-server-name`, `--tls-server-enabled`.
